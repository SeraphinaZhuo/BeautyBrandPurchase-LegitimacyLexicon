# 数据说明

## 数据可获得性

本仓库**不包含**发言原文级数据。原始的微博与小红书文本、清洗后的样本、以及标注工作文件均不纳入版本控制，原因是：

* 这些数据包含可追溯到单条公开发言的原文，涉及平台服务条款与用户隐私；
* 依据 METHODS.md 第 1 节的分析原则，本项目仅公开代码、词典、清洗规则与聚合统计结果，不默认公开原文级数据。

仓库中公开的是：全部处理脚本、词典、编码手册、方法说明，以及不含原文的聚合统计报告（见 `reports/`）。凭借这些内容，配合符合下述字段结构的数据，第三方可以完整复现本项目的处理流程。

## 数据来源与采集口径

数据为 2023 年 7 月至 11 月期间，围绕特定美妆与日化品牌、与日本核污染水排海事件相关的社交媒体公开发言。采集按品牌别称与危机/替代词簇的组合进行关键词检索，覆盖三个时段（事件前 / 爆发期 / 冷却期）与五个品牌（资生堂、SK-II、花王、珀莱雅、蜂花）。采集与清洗口径的完整说明见 METHODS.md 第 2 节。

## 字段结构

### 微博数据（清洗前）

| 字段 | 含义 |
|---|---|
| `platform` | 平台来源 |
| `post_id` | 发言唯一 ID |
| `parent_post_id` | 若为评论，其挂靠的主贴 ID；主贴则为空 |
| `brand_category` | 涉及的品牌 |
| `time_period` | 时段（T0 事件前 / T1 爆发期 / T2 冷却期） |
| `date` | 发布日期（YYYY-MM-DD） |
| `content_type` | 数据类型（主贴正文 / 热门评论） |
| `text_content` | 发言文本 |
| `likes` / `reposts` / `comments_count` | 点赞数 / 转发数 / 评论数 |
| `keyword_hit` | 触发采集的关键词 |
| `keyword_cluster` | 触发采集的关键词簇 |
| `user_type` | 账号类型（普通用户 / 蓝V认证账号 / 品牌官方） |
| `user_id` | 用户 ID |

### 微博数据（清洗后新增字段）

清洗脚本 `clean_weibo.py` 在上述字段基础上增加以下列：

| 字段 | 含义 |
|---|---|
| `text_norm` | 规范化后的文本（HTML 转义还原、空白折叠） |
| `crisis_flag` | 是否包含核污水相关词簇（0 / 1） |
| `huaxizi_flag` | 蜂花样本中是否受同期花西子事件影响（0 / 1） |
| `guohuo_flag` | 是否包含国货相关表达（0 / 1） |

### 小红书数据

主贴字段：`平台来源`、`帖子ID`、`品牌`、`发布时间`、`数据类型`、`具体文本`、`点赞数`、`搜索关键词`。

评论字段：`平台来源`、`帖子ID`、`品牌`、`评论用户名`、`评论时间`、`数据类型`、`具体文本`、`评论点赞数`、`抓取关键词`。

## 复现方式

准备符合上述字段结构的微博数据后，依次运行：

```
python .\scripts\clean_weibo.py --input <微博原始数据> --outdir reports/cleaned_weibo
python .\scripts\audit_seed_words.py --seeds .\lexicon\seed_words_v0.csv --weibo <清洗后的微博数据> --xhs-notes <小红书笔记数据> --xhs-comments <小红书评论数据> --outdir .\reports\audited
```

输出的统计报告应与 `reports/` 目录下已公开的结果一致。跨环境一致性的验证记录见 VERIFICATION.md。
